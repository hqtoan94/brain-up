# {{NAME}}

Standalone personal brain repository (plain Markdown, portable, greppable).

## Structure

```
.
├── index.md            GENERATED map of the brain (scripts/regen-indexes)
├── scripts/            no-AI automation: lint, regen-indexes, freshness, digest
├── raw/                immutable sources — agent reads, never writes
└── brain/              the working vault — agent reads and writes here
    ├── inbox/          quick captures — process weekly
    ├── projects/       things with a deadline (PARA)
    ├── areas/          ongoing responsibilities (PARA)
    ├── resources/      atomic + distilled knowledge notes
    ├── archive/        done / inactive (PARA)
    ├── goals/          vision + yearly outcomes
    ├── journal/        one file per day
    ├── people/         contacts & meeting notes
    ├── career/         portable, anonymized career record
    └── templates/      starters for each note type
```

## Setup

Install / sync this brain on a machine with [brain-up](https://github.com/hqtoan94/brain-up).

## Conventions

- Every note has YAML frontmatter with `type`, `title`, `created`, `updated`, `tags`.
- `index.md` files (root and per-folder) are generated by `scripts/regen-indexes` — never hand-edited.
- Notes in `brain/resources/` declare `maturity: atomic | distilled`.

## Daily usage

- capture quick thoughts to `brain/inbox/`
- journal in `brain/journal/`
- organize into `brain/projects/`, `brain/areas/`, `brain/resources/` during review

## How the agent works

This brain is operated by the **second-brain** sub-agent at
[`.claude/agents/second-brain.md`](.claude/agents/second-brain.md) — the source of truth for
layout, conventions, the work-vs-personal boundary, and sensitivity rules. Trigger phrases
(capture, journal, process inbox, weekly review, search, ingest, lint, …) route to skills in
[`.claude/skills/`](.claude/skills/). Quick lookups live in [`CHEATSHEET.md`](CHEATSHEET.md).

# Career

The portable, **anonymized** record of my career. One file per role.

This is what I take with me when I leave a job — the distillate, not the deliverables. Written (or finalized) on the way out, using the exit ritual.

## What lives here

- Role summaries: title, dates, level, scope, generalized wins.
- Lessons I learned **as a professional** — principles, patterns, anti-patterns.
- Skills that genuinely grew at this job (logged inside the relevant project or area while you were building them).
- Anonymized story library — narrative versions of wins for interviews / reflection, using placeholders (`$COMPANY`, `$PROJECT_X`).

## What never lives here

- Employer / client / customer names that identify the business relationship.
- Product or internal tool names.
- Numbers that reveal confidential data (revenue, headcount, costs, timelines tied to unreleased work).
- Verbatim quotes from colleagues; comp or review content.

If any of the above would help you remember the *learning*, rewrite the sentence until it no longer reveals the source.

## Filename

`YYYY-YYYY-role-slug.md`, where the range is start year to end year (or `-present` for current).

Examples:
- `2021-2024-senior-engineer.md`
- `2024-present-staff-engineer.md`

Use the same pattern across jobs — do **not** include the company name in the filename.

## When to write this

- **Month 6 at a new role** — a first draft of scope, wins so far, skills growing. Thin is fine.
- **Quarterly** — append notable wins and lessons as they happen.
- **On exit** — final pass via the exit ritual (agent can drive). This is when the file gets its durable shape.

## Using this for interviews / resumes

When prepping for a conversation about your career, read these files first. They're already anonymized and already in the language of outcomes — not internal jargon.
